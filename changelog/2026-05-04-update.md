## 架构级 / 新模块

### `/goal` — 持久化跨轮目标（Ralph Loop）

新模块 `hermes_cli/goals.py`（535 行）实现一个**辅助模型 judge** 驱动的目标循环：

- 每个 turn 完成后，judge 判定目标是否达成
- 未达成则把"continuation prompt"自动入队，**普通 user message**，不改 system prompt、不切 toolset、**保持 prompt cache**
- judge fail-OPEN，turn 预算（默认 20）兜底
- 用户新消息抢占；`/resume` 自动恢复（state 存 SessionDB `state_meta`）
- CLI 子命令：`/goal <text>` / `status` / `pause` / `resume` / `clear`

详见 [[persistent-goals]]。

提交：`265bd59` `feat: /goal — persistent cross-turn goals (Ralph loop) (#18262)`，`d87fd9f` `fix(goals): make /goal work in TUI and fix gateway verdict delivery (#19209)`，`cf2b2d3` 文档页面，`0b76d23` llms.txt 收录。

### Kanban — 多 Profile 协作看板

新模块 `hermes_cli/kanban_db.py`（2765 行）+ `hermes_cli/kanban.py`（1393 行）+ `tools/kanban_tools.py`（726 行）：

- **profile-agnostic** SQLite DB（`$HERMES_HOME/kanban.db`），多 Profile 共享同一看板
- 并发：WAL + `BEGIN IMMEDIATE` + CAS（`tasks.status` / `tasks.claim_lock`）；SQLite WAL lock 序列化 writer，无重试循环、无分布式锁
- 5 张表：tasks / task_runs（重试历史）/ task_links / task_comments / task_events / kanban_notify_subs（gateway 闭环订阅）
- workspace 解耦 git：`scratch` / `worktree` / `dir:<path>`
- 7 个结构化 worker tool（kanban_show/complete/block/heartbeat/comment/create/link），仅在 `HERMES_KANBAN_TASK` 环境变量存在时注册——普通 chat 看不到
- claim TTL 默认 15 分钟，worker 心跳续约
- 新 skill：`kanban-orchestrator` / `kanban-worker` / `kanban-video-orchestrator`（creative）

详见 [[kanban-collaboration-board]]。提交：`c868425` `feat(kanban): durable multi-profile collaboration board (#17805)`，`97d6f25` 测试断言更新。

### Tool Loop Guardrails — 工具调用循环护栏

新模块 `agent/tool_guardrails.py`（455 行），**warning-first** 设计：

- 三种 loop 模式检测：
  1. **Exact failure**：同 `(tool_name, sha256(canonical_args))` 重复失败（warn≥2，block≥5）
  2. **Same-tool failure**：同工具任意 args 累计失败（warn≥3，halt≥8）
  3. **Idempotent no-progress**：只读工具同结果（warn≥2，block≥5）
- 默认 `warnings_enabled=true, hard_stop_enabled=false`——只温和提示，不打断 agent
- mutating vs idempotent tool 集合写死：`MUTATING_TOOL_NAMES` / `IDEMPOTENT_TOOL_NAMES`
- 决策从不暴露 raw args（只 hash）
- 失败检测复用 `agent.display._detect_tool_failure`，保证 CLI `[error]` 标签和护栏判断一致

详见 [[tool-loop-guardrails]]。提交：`58b8996` `fix(agent): add tool-call loop guardrails`，`0704589` `fix(agent): make tool loop guardrails warning-first`，`8fa44b1` `fix(guardrails): preserve display _detect_tool_failure semantics`。
