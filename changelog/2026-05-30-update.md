## 6. `read_file` 紧凑行号 gutter ~14% token 节省（#35368）

- **commit**：`ea6eaabd8 perf(read_file): compact line-number gutter — ~14% fewer tokens per read (#35368)`，2026-05-30
- 旧 gutter：`"     1|content"`（固定宽度 zero/space padding）
- 新 gutter：`"1|content"`（紧凑形式）

**实测**（cl100k tokenizer，真实 Hermes 源码）：

| 形态 | 相对成本 |
|------|---------|
| padded（旧默认） | 基线 |
| 无行号 | -33% |
| compact `<n>\|`（新默认） | **-14%** |

> padded vs compact 在"行引用 / patch / 值查找 / 结构理解"4 类任务（Sonnet 4.6，A/B 2 pass，每条断言对 ground truth）都是 4/4 pass；**完全去掉行号** regress 到 3/4（模型靠手数行算错）。所以**保留行号，砍掉 padding**。

**实现**（`tools/file_operations.py:705-730 _add_line_numbers`）：

```python
# HERMES_READ_GUTTER=padded 恢复旧固定宽度行为
padded = (_os.environ.get("HERMES_READ_GUTTER") or "").lower() == "padded"
```

**下游影响 = 零**：`patch` / `fuzzy_match` 永不消费 gutter（它们匹配 `old_string` 文本、内部算字符 offset）；无下游 parser 依赖固定列宽。

测试：`tests/tools/test_file_operations.py` / `test_file_operations_edge_cases.py` 更新 3 处格式断言到 compact，新增 env-override 测试覆盖 `HERMES_READ_GUTTER=padded` 回归。209 个 file-tool 测试全绿。

## 7. `write_file` / `patch` 原子写（#35252）

- **commit**：`39f6b6e9d fix(file-tools): make write_file/patch atomic (temp-file + rename) (#35252)`，2026-05-30

**问题**：旧 `write_file` 直接 `cat > path` 流式写入 —— crash / SIGKILL / 管道断在中途，目标文件留半写损坏状态。`patch_replace` 走 `write_file`，受同一缺陷。

**修复**：内容流入**同目录** temp file，然后 `mv` 覆盖目标 —— same-filesystem rename，在 POSIX 与全部 terminal backend（local / docker / ssh / modal）上都是**原子**操作。

- 写失败留**原文件**字节完整，不留 temp leak
- 文件 mode 跨 swap 保留（stat + chmod，GNU/BSD 兼容）
- 内容仍走 stdin，**无 ARG_MAX 限制**
- 任何错误路径都有 trap 清理 temp

测试：`tests/tools/test_file_write_safety.py`（76 行新增 `TestAtomicWrite`，真实 LocalEnvironment 无 mock），覆盖 inode-change-on-overwrite / mode preservation / failed-write-leaves-original / no-temp-leak / 特殊字符 / patch routing。同时把 `test_file_operations.py` 里 2 个 `cat >` 字面量断言改为 `stdin_data` 行为信号断言。200 个 file-tool 测试全绿。

**关联 fix**：

- `5f84c9144 fix(file-tools): handle UTF-8 BOM in read_file / write_file / patch (#35278)` —— BOM 现在被正确识别和保留
- `96643b4a5 fix(file-tools): anchor relative-path resolution to absolute base; report resolved path (#35399)` —— 相对路径定位到 absolute base，错误消息显示解析后路径

## 9. Session model 中途切换持久化（fixes #34850）

- **commit**：`794519c6a fix(state): persist mid-session model switch to database`，2026-05-30
- 跟进测试：`e1945ff69 test(state): cover update_session_model overwrite + getattr-guard text path`

**bug**：用户 mid-session 走 `/model` 切换，gateway 更新 in-memory agent 和 session override，**但 DB 从未更新**。`update_token_counts()` 里 `COALESCE(model, ?)` 只填 NULL，所以 dashboard 永远显示原始 model。

**修复**（`hermes_state.py:968-980`）：

```python
def update_session_model(self, session_id: str, model: str) -> None:
    """Update the model for a session after a mid-session switch.

    Unlike ``update_token_counts`` which uses ``COALESCE(model, ?)``
    (only filling in NULL), this unconditionally sets the model column
    so that the dashboard reflects the user's latest /model choice.
    """
    def _do(conn):
        conn.execute(
            "UPDATE sessions SET model = ? WHERE id = ?",
            (model, session_id),
        )
    self._execute_write(_do)
```

`gateway/run.py` +29 行：interactive picker 与直接 `/model` 命令两条路径都调用。

## 10. Starlette CVE-2026-48710 BadHost 修复 pin（#35118）

- **commit**：`0437137ff security: pin patched Starlette (>=1.0.1) for CVE-2026-48710 BadHost (#35118)`，2026-05-29

**漏洞**：Starlette < 1.0.1（CWE-444）—— HTTP Host header 在用于重建 `request.url` 前未做校验。畸形 Host 会让 `request.url.path` 与 ASGI 路由真正分发的原始路径**脱钩**。基于 `request.url`（而非 `scope["path"]`）做 path-based authorization 的 middleware/endpoint 可被绕过。

**Hermes 仅经 transitive 引用 Starlette**：

```text
[web]          → fastapi==0.133.1     (starlette>=0.40.0, no upper bound)
[mcp]          → mcp==1.26.0 + sse-starlette
[computer-use] → mcp==1.26.0
[dev]          → mcp==1.26.0
```

无 upper bound，新解析能落到任何 pre-1.0.1 release。

**修复**：在每个暴露 Starlette-backed server surface 的 extra 里**直接 pin** `starlette==1.0.1`，并重新生成 `uv.lock`（hash-verified）。

`pyproject.toml` 中的 pin 点：

| 行 | extra | 说明 |
|----|-------|------|
| `:86 dev` | `mcp==1.26.0, starlette==1.0.1` | CVE-2026-48710 |
| `:118 mcp` | `mcp==1.26.0, starlette==1.0.1` | CVE-2026-48710 |
| `:125 computer-use` | `mcp==1.26.0, starlette==1.0.1` | CVE-2026-48710 |
| `:180 web` | `fastapi==0.133.1, uvicorn[standard]==0.41.0, starlette==1.0.1` | CVE-2026-48710 |

`tools/lazy_deps.py:176 "starlette==1.0.1"` —— 镜像 `tool.dashboard` lazy-install map，确保 on-demand dashboard install 不会 re-resolve 到 vulnerable 版本。

## 11. TUI `/agents` 看板 nudge（#34704 + #35216）

- **commit**：`5a72e82fd feat(tui): nudge toward /agents dashboard when delegation starts`，2026-05-29
- 跟进：`9d2571c86 fix: surface /agents nudge while delegate_task is in-flight (TUI + CLI)`，2026-05-30

**痛点**：TUI 早就有富 `/agents` spawn-tree 看板（live tree / timeline / per-child tokens·cost·files·tools / kill·pause），但**没有任何 surface 提示用户**它存在 —— delegation 期间 transcript 静默，只有"已经会"`/agents` 的用户用得上。

**修复**：第一次 turn 触发 delegation 时，落一条**一次性 transient activity hint**：

```text
subagents working · /agents to watch live
```

风格对齐既有的"· /logs to inspect"。

**guard**（`ui-tui/src/app/createGatewayEventHandler.ts:143-186`）：
- 每 turn 最多一次（`message.start` 时重置）
- `/agents` overlay 已开则静默
- `display.tui_agents_nudge` config 开关（default true，关闭 → false）

**config 默认**（`hermes_cli/config.py:1221-1230`）：

```python
# When true (default), `hermes --tui` drops a one-time hint
# ("subagents working · /agents to watch live") the first time a turn
# starts delegating, nudging the user toward the live spawn-tree
# dashboard. Set false to suppress the hint.
"tui_agents_nudge": True,
```

**事件挂在 `subagent.start`**，不挂 `subagent.spawn_requested` —— `tools/delegate_tool.py` 的 progress callback 只把 start/complete 转给 gateway，spawn_requested 被 drop，所以 start 才是 TUI 第一个可靠拿到的 delegation 事件。spawn_requested 同样接线但 guarded once-per-turn，留给未来。

**第二段 fix #35216 / #34704**：

- TUI（`ui-tui/src/components/thinking.tsx`）：任何 "Delegate Task" 工具组**一出现就显示 `(/agents)`**，不再等 subagent 已注册
- CLI 侧：单次 `delegate_task` 进行中（不只是多个 subagent）也 surface nudge
